## 1. 배경 및 문제 정의

한국에는 국가기술자격 500여 종, 국가전문자격 수십 종, 국가공인 민간자격 100여 종이 존재한다. 각 자격증의 원서접수·시험·발표 일정은 시행기관(Q-Net, 대한상공회의소, 금감원, 국사편찬위원회 등) 사이트에 분산되어 있고, 회차별로 갱신된다.

수험생이 겪는 문제:

1. 관심 자격증의 일정을 매번 여러 사이트에서 직접 확인해야 한다.
2. **원서접수 시작일을 놓쳐 다음 회차까지 기다리는 사례**가 빈번하다 (특히 인기 종목은 조기 마감).
3. 합격자 발표일을 잊고 지나치거나, 합격 후 실기 접수 일정을 놓친다.
4. 여러 자격증을 병행 준비할 때 일정 충돌·우선순위 파악이 어렵다.

## 2. 제품 비전

> 모든 한국 국가공인 자격증의 일정을 한 곳에서 보고, 놓치지 않도록 알려주는 서비스.

## 3. 목표 (Goals & Non-goals)

### 3.1 Goals

- 한국 국가공인 자격증 전체의 회차별 일정(접수·시험·발표)을 정확하게 표시한다.
- 사용자가 관심 자격증을 구독하면 주요 일정마다 자동으로 알림을 받는다.
- 일정 데이터가 변경되면 (연기·취소·일정 변경) 빠르게 반영하고 재알림한다.

### 3.2 Non-goals (MVP 범위 외)

- 수험서·인강 판매, 학습 콘텐츠 제공
- 합격 후기·커뮤니티 기능
- 자격증 추천 알고리즘 (관심사 기반 자동 추천)
- 모바일 네이티브 앱 (MVP는 반응형 웹)
- 결제·유료 기능

## 5. 핵심 사용자 시나리오

### S1. 관심 자격증 등록
1. 사용자는 첫 방문 시 회원가입(이메일 또는 소셜 로그인).
2. 자격증 카테고리에서 "정보처리기사"를 검색·선택해 즐겨찾기 등록.
3. 알림 설정 모달에서 "접수 시작 3일 전, 접수 마감 1일 전, 시험일 1주 전, 발표일 당일"을 선택.

### S2. 일정 확인
1. 메인 대시보드 진입 시, 즐겨찾기한 자격증의 다가오는 일정이 카드 + 캘린더 뷰로 표시.
2. 카드 클릭 시 회차별 상세(접수/시험/발표 일자, 응시료, 공고 원문 링크) 확인.

### S3. 알림 수신
1. 접수 시작 3일 전 오전 9시, 사용자에게 이메일 + 웹 푸시 알림 발송.
2. 알림 클릭 시 해당 자격증 상세 페이지 + Q-Net 접수 URL로 바로 이동.

### S4. 일정 변경 대응
1. 시행기관에서 시험 일정을 변경하면, 운영자/크롤러가 데이터 갱신.
2. 해당 자격증 구독자 전원에게 "일정 변경" 알림 자동 발송.

## 9. UX 와이어프레임 개요

- **랜딩 페이지**: 검색바 + 인기 자격증 + "지금 다가오는 시험" 미리보기
- **자격증 상세**: 상단 요약 / 회차별 타임라인 / 알림 설정 버튼 / 공식 공고 링크
- **마이 대시보드**: 좌측 즐겨찾기 리스트, 우측 다가오는 일정 카드 + 월간 캘린더 토글
- **알림 설정**: 자격증별 알림 시점 토글 + 채널 선택 (이메일/웹푸시)
- **알림 인박스**: 받은 알림 이력, 미확인 표시

## 10. 기술 스택 (제안)

- **Frontend**: Next.js (App Router), TypeScript, TailwindCSS, shadcn/ui, TanStack Query
- **Backend**: Node.js (NestJS) 또는 Python (FastAPI)
- **DB**: PostgreSQL (메인) + Redis (캐시·알림 큐)
- **알림**: SendGrid/AWS SES (이메일), Web Push API + VAPID
- **크롤러**: Python (Scrapy/Playwright), 스케줄러는 GitHub Actions 또는 Airflow
- **인프라**: AWS (ECS Fargate + RDS + S3 + CloudFront) 또는 Vercel + Supabase
- **모니터링**: Sentry, Datadog, Slack 웹훅

## 13. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| 시행기관 사이트 구조 변경으로 크롤러 깨짐 | 일정 데이터 누락 → 잘못된 알림 | 변경 감지 모니터링 + 관리자 수동 보정 |
| 일정 오기재로 인한 사용자 불이익 | 신뢰도 하락 | 모든 일정에 "공식 공고 링크" 병기, 면책 고지 |
| 크롤링 법적 이슈 | 서비스 중단 | robots.txt 준수, 공개 정보만 수집, 필요 시 기관 협의 |
| 알림 스팸 인식 | 구독 해지 증가 | 사용자가 알림 시점·채널 세부 제어, 다이제스트 옵션 |
| 개인정보 (PIPA) | 법적 리스크 | 이메일만 최소 수집, 알림 발송 외 활용 금지, 약관·동의 명시 |

## 14. 오픈 이슈 / 추가 결정 필요 사항

- 수익 모델: 무료 + 광고 vs 프리미엄(다중 알림 채널, 캘린더 연동) vs 광고 없는 운영
- 카카오톡 알림톡 도입 시기 (비용 부담 큼)
- 자격증 마스터 데이터 초기 구축 범위 (상위 50종 vs 100종 vs 전체)
- 어학 시험 (TOEIC, TOEFL 등) 포함 여부 — 자체 사이트가 있고 정기 시행이라 우선순위 낮을 수 있음
